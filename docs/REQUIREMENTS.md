# 桌面端数据库客户端工具 — 需求文档
## 1. 项目背景与目标
### 1.1 背景
市面上主流数据库管理工具（如 Navicat、DBeaver）功能强大但存在界面复杂、体积臃肿、部分收费较贵等问题。本项目旨在开发一款**轻量、简洁、跨平台**的数据库客户端工具，聚焦核心高频功能，降低使用门槛。
### 1.2 产品定位
- 一款面向开发者/运维/数据分析人员的桌面端数据库连接与管理工具
- 对标 Navicat 的核心能力，但界面更简洁、启动更快、学习成本更低
- 第一版聚焦 **MySQL、Oracle、Redis** 三类主流数据库
### 1.3 目标用户
- 后端开发工程师（日常查询、调试 SQL）
- 运维/DBA（数据库管理、监控）
- 数据分析人员（数据查看、导出）
---
## 2. 支持的数据库类型（V1 范围）
| 数据库 | 连接方式 | 优先级 |
|---|---|---|
| MySQL | TCP/IP，支持 SSH 隧道 | P0 |
| Oracle | TNS / Service Name / SID | P0 |
| Redis | 单机 / 哨兵 / 集群模式 | P0 |
> 后续可扩展 PostgreSQL、SQL Server、MongoDB 等（V2 规划）。
---
## 3. 核心功能需求
### 3.1 连接管理
- 新建/编辑/删除/复制数据库连接
- 连接分组管理（文件夹形式，支持拖拽排序）
- 连接测试（Test Connection）功能，实时反馈成功/失败原因
- 支持保存密码（本地加密存储）或每次手动输入
- 支持 SSH 隧道连接（跳板机场景）
- 支持连接超时、重连机制配置
- 最近连接历史记录
### 3.2 数据库对象浏览（MySQL / Oracle）
- 树形结构展示：数据库 → 表 / 视图 / 存储过程 / 函数 / 触发器 / 索引
- 表结构查看：字段名、类型、长度、是否主键、是否为空、默认值、注释
- 支持右键菜单：新建表、修改表、删除表、清空表、导出表结构
- 表数据分页浏览（默认每页 1000 行，可配置）
- 支持按列排序、简单筛选（不写 SQL 即可筛选）
### 3.3 SQL 编辑器
- 语法高亮（区分 MySQL / Oracle 方言）
- SQL 自动补全（表名、字段名、关键字）
- 多标签页同时编辑多个 SQL 窗口
- 执行单条 / 执行全部 / 执行选中语句
- 查询结果以表格形式展示，支持排序、复制、导出
- 执行历史记录（可回溯查看过去执行过的 SQL）
- 查询结果直接编辑并生成 UPDATE 语句（类 Navicat 的“网格编辑”功能）
- SQL 格式化（美化）功能
### 3.4 数据编辑
- 表格内直接编辑单元格（双击编辑）
- 新增行 / 删除行 / 批量删除
- 编辑后需点击“提交”才真正写入数据库（防误操作）
- 支持 NULL 值的显式设置
### 3.5 Redis 专属功能
- Key 浏览（支持按 pattern 搜索、分页加载，避免 KEYS * 阻塞）
- 支持 String / Hash / List / Set / ZSet 五种类型的可视化查看与编辑
- TTL 查看与设置
- 支持切换 DB（0-15）
- 支持哨兵模式和集群模式连接
- 简易命令行模式（可直接输入 Redis 命令执行）
### 3.6 导入导出
- 数据导出：CSV、Excel、SQL、JSON
- 数据导入：CSV、Excel、SQL 文件执行
- 表结构导出为 SQL 建表语句
- 支持导出查询结果（而不仅仅是整表）
### 3.7 其他辅助功能
- 多窗口/多标签页管理
- 快捷键支持（执行 SQL、保存、切换标签页等，尽量兼容 Navicat 习惯）
- 深色/浅色主题切换
- 连接信息本地加密存储，支持导出/导入连接配置（便于团队共享，不含密码明文）
---
## 4. UI/UX 设计要求（简洁风格）
- **整体风格**：扁平化、留白充分，避免堆砌图标和多层工具栏
- **布局**：左侧连接树 + 中间主工作区（Tab 切换）+ 底部状态栏（显示执行耗时、影响行数等）
- **减少弹窗**：能用侧边栏/内嵌面板解决的，不用弹窗
- **一致性**：MySQL、Oracle、Redis 的操作交互逻辑尽量统一，降低用户学习成本
- **响应速度优先**：大数据量表格采用虚拟滚动，避免卡顿
- 支持窗口大小记忆、面板拖拽调整宽度
---
## 5. 非功能性需求
| 类别 | 要求 |
|---|---|
| 性能 | 应用启动时间 < 3 秒；万级数据分页加载 < 1 秒 |
| 安全 | 密码本地加密存储（如使用系统 Keychain/DPAPI）；支持连接信息脱敏导出 |
| 兼容性 | 支持 Windows 10+、macOS 11+（可选支持 Linux） |
| 稳定性 | 单个连接异常不影响其他已开启连接/标签页 |
| 可维护性 | 数据库驱动层与 UI 层解耦，便于后续扩展新数据库类型 |
---
## 6. 技术选型建议（可选参考）
- **客户端框架**：Electron + React / Vue（跨平台桌面应用，生态成熟）或 Tauri（体积更小、性能更好，Rust 后端）
- **MySQL 驱动**：mysql2 (Node.js) 
- **Oracle 驱动**：node-oracledb（需依赖 Oracle Instant Client）
- **Redis 驱动**：ioredis
- **SQL 编辑器组件**：Monaco Editor（VS Code 同款编辑器内核）
- **本地存储**：SQLite（存储连接配置、历史记录等元数据）
---
## 7. 版本规划建议
| 版本 | 内容 |
|---|---|
| V0.1（MVP） | MySQL 连接、SQL 编辑器、基础表数据浏览与编辑 |
| V0.5 | 增加 Oracle 支持、数据导入导出 |
| V1.0 | 增加 Redis 支持、SSH 隧道、连接分组、主题切换 |
| V2.0（规划） | PostgreSQL、SQL Server 支持，团队协作（连接共享） |
---
## 8. 风险与待确认事项
- Oracle 驱动依赖 Instant Client，安装分发方式需提前设计（是否内置）
- 是否需要支持多人协作（连接配置云同步）？
- 是否需要国际化（中英文切换）？
- 密码存储方案需与安全团队/负责人确认加密强度要求
- 是否需要命令行工具（CLI）配套？
